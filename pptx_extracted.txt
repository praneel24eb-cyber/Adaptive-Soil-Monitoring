4 1. Overview of the Topic Soil fertility plays a crucial role in determining crop yield and overall agricultural productivity. It depends on key parameters such as nitrogen (N), phosphorus (P), potassium (K), moisture, temperature, and electrical conductivity, which continuously vary due to environmental and farming conditions. Traditional soil monitoring methods rely on periodic testing or simple sensor readings, which provide only static information about soil conditions. These approaches do not capture how soil fertility changes over time and do not provide insights into whether corrective actions, such as fertilizer application, are effective. This project introduces an Adaptive Soil Fertility Monitoring System using Edge Intelligence, which performs real-time sensing, trend analysis, and classification directly on an embedded device. By combining IoT, statistical methods, and machine learning, the system provides meaningful insights into soil health and supports informed agricultural decision-making. Introduction

8 Problem Definition Context of the Problem A large portion of agriculture depends on manual decision-making for fertilizer usage. Soil testing methods are often infrequent and delayed, making timely action difficult. Modern IoT systems generate data but provide limited intelligent interpretation. Variability in soil conditions across locations makes uniform analysis ineffective. Significance Of Addressing the Problem Enables better soil health management through continuous monitoring. Supports data-driven fertilizer decisions, reducing resource wastage. Improves agricultural productivity and efficiency. Contributes to sustainable farming practices by optimizing input usage.

10 Methodology

--- Slide 11 ---
Defining the overall system architecture for an edge-based soil monitoring system that integrates sensing, processing, and decision-making into a unified pipeline Acquiring real-time soil data such as NPK, moisture, temperature, and EC using an RS485-based sensor deployed in the field Transferring sensor data to the ESP32 microcontroller for local processing, ensuring low latency and independence from cloud infrastructure Performing preprocessing and feature extraction by normalizing sensor values and structuring inputs for further analysis Applying the CUSUM drift detection algorithm to identify gradual changes and trends in soil fertility over time Implementing adaptive calibration to dynamically adjust baseline values based on site-specific soil conditions Running a machine learning model on the ESP32 to classify soil fertility into meaningful categories Logging fertilization events and tracking changes in nutrient levels to analyze fertilizer effectiveness over time Displaying results locally on an OLED and transmitting processed insights via MQTT to a dashboard for monitoring 11 Methodology

--- Slide 12 ---
12 Methodology Data Analysis Techniques: Preprocessing techniques such as normalization and feature scaling to ensure consistent and reliable input data Drift detection using the CUSUM algorithm to identify early changes in soil nutrient levels Trend-based analysis to understand variations in soil parameters over time Machine learning-based classification to convert sensor data into actionable fertility insights System Intelligence Adaptive calibration mechanism that automatically adjusts to variations in soil characteristics across locations Event-driven fertilization tracking to monitor post-application nutrient response and effectiveness Context-aware decision-making using both raw sensor inputs and derived features like drift score

--- Slide 13 ---
13 Methodology Tools and Technologies Used Hardware: ESP32 microcontroller acts as the core processing unit, interfacing with the RS485 soil sensor via MAX485 for reliable acquisition of NPK, moisture, temperature, and EC values Communication: MQTT protocol is used for lightweight and efficient transmission of processed insights and alerts, minimizing bandwidth usage and enabling real-time monitoring ML Tools: scikit-learn is used for training the machine learning model offline, while m2cgen converts the trained model into optimized C code for deployment on ESP32 Dashboard: Node-RED manages data flow, InfluxDB stores time-series sensor data, and Grafana provides interactive dashboards for visualization of trends and system outputs Development: Arduino IDE or PlatformIO is used for firmware development, debugging, and integration of all modules into a complete embedded system
